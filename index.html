Confused or curious about any aspect of the 2048 Power Compendium? Read your answers here!

Basics of the 2048 Power Compendium

The 2048 Power Compendium is a collection of 100 different variations of the game 2048. You've probably already played, or at least heard of, 2048, but if you haven't, here's a summary:

A game of the original 2048 in the 2048 Power Compendium

2048 is a game that's played on a 4x4 grid of tiles. On each move, you choose one of the four orthogonal directions, and all the tiles slide in that direction until they hit another tile or the edge of the grid. The tiles are solid to each other, but if two tiles of the same number collide, they merge into a single tile with twice the number: two 1s merge into a 2, two 2s merge into a 4, two 4s merge into an 8, and so on. After each move, a new tile spawns. The game continues until the grid is full and no more merges can occur, at which point you're out of moves and the game is over. You win by reaching the 2048 (211) tile, but if you're good enough at the game, it's possible to get even higher tiles.

A game of 2187, which uses powers of 3 instead of powers of 2

There are many 2048 variants out there on the internet - what makes the 2048 Power Compendium special is that it's a collection of a bunch of variants, and while a few of the variants originated elsewhere, most of them are unique to the 2048 Power Compendium. When you first open the 2048 Power Compendium, you're looking at the menu screen, which contains the first 25 modes. Click on one of the tiles on the grid there to go to a screen that explains what that mode is. On that screen, you can change the dimensions of the grid, then you can begin a game in that variant by clicking the "Start Game" button. Each mode has different merging rules - the later modes tend to be more complicated than the earlier ones, so it's recommended that (at least for the main modes) you start at the top-left and work your way across the different modes from left to right, top to bottom (like how you read words on a page).

Note that there are a few differences between the 2048 Power Compendium and other 2048 variants. Unlike most 2048 versions, the 2048 Power Compendium does not have an automatic game over check, but if you try to move in a direction and nothing happens, that direction goes black until the next successful move, and it's game over once all directions have failed. The 2048 Power Compendium supports arrow key controls, WASD controls, and clicking the direction buttons to move, but unlike other 2048 versions, swiping the screen is not supported. If you feel that the movement animation is too fast or too slow, then click "Go to Modifiers" on the menu and change the animation speed there (It's recommended that you don't alter any of the other modifiers other than Animation Speed Multiplier until you've played a few games and gotten used to the various modes). Progress isn't automatically saved, but you can save your progress yourself by clicking "Get save code" in the middle of a game and copying the code there, then clicking "Resume From Save Code" on the menu and pasting that save code there when you want to resume the game. Also, upon finishing a game, there's an option to get a save code to view a replay of that game.

If you have any other questions, then read on. Otherwise, click "Return to menu" and enjoy the 2048 Power Compendium!

Modes

The first 25 modes of the 2048 Power Compendium

Each mode of the 2048 Power Compendium is a 2048 variant with different merge rules, and different tiles that arise from these merge rules. Most of the modes are based on powers of a certain number: 2048 uses powers of 2, 2187 uses powers of 3, 1024 uses powers of 4, and so on. A few modes use powers of a certain number offset somehow, such as 2047 (powers of 2, minus 1) and 2059 (powers of 3, minus powers of 2). 5040 and XXXX are special, as both of them combine previous modes or powers somehow. Unlike 2048, most modes have "intermediate" tiles: for example, in 2187, there's a tile for each power of three, but also for double each power of three, which means the possible tiles are 1, 2, 3, 6, 9, 18, 27, 54, 81, 162, 243, and so on. Each mode has a default grid size, but feel free to play on a differently-sized grid (by clicking the plus and minus buttons next to the grid width and height on the gamemode explanation screen) if you find the default grid size for that mode too hard or too easy.

Once you've played some of the regular modes and have a handle on the differences between each mode, you can click the "Switch Pages" button on the menu. This lets you switch between the four pages of modes, each of which has 25 modes, for a total of 100. The four pages each have a different overall "feel" to them: Page 1 consists of the standard modes centered around powers of numbers (which is, at least in theory, the central theme of the Power Compendium), Page 2's modes also play in a standard 2048 fashion but don't have as much "powers of numbers" focus (and some of the later ones are a little different in gameplay to a standard 2048 mode), Page 3's modes are more open-ended modes with multiple pathways of possible tiles (and, in some cases, different goals entirely than getting to particular tiles), and Page 4's modes are defined by various weird gimmicks that set them apart from the rest of the modes.

Not all of the 2048 Power Compendium modes will be equally fun - try out various modes and find your favorites!

Mode Modifiers

The mode explanation screen for XXXX

In addition to allowing you to change the grid size, some of the more advanced modes have "mode modifiers", options that let you change specific aspects of that mode, often relating to the exact details of the merge rules or the spawning tiles. In particular, several modes (many of which are on Page 3) include a "random goals" modifier, since those modes often have enough possible tiles that you may want to aim for tiles other than the given win conditions when replaying that mode. Some of these mode modifiers are changed by clicking plus/minus buttons, some of them are changed by clicking "Switch this Modifier" buttons, and some of them are changed by entering their new value into the provided textbox.

Gameplay of XXXX with the "Discovered Tiles" box clicked to alter its appearance

I didn't know where else in the guide to put this, so I'll mention it here. Some modes have more statistic boxes alongside the score, such as showing how many different tiles you've seen in this round. If one of these statistic boxes has a border around it, that means that box can be clicked to alter its display: for example, Discovered Tiles boxes can be clicked to change them between showing the amount of discovered tiles and showing all of the discovered tiles individually.

Modifiers

If the 100 modes on their own aren't enough for you, then click "Go to Modifiers" on the menu to access a bunch of different modifiers you can change. These modifiers affect EVERY mode (for the most part), and they allow a pretty good degree of customization. The Modifiers selection screens are split up into six pages - to switch between them, click the blue and orange arrows. With the exception of the Animation Speed Multiplier modifier (which is just aesthetic, and you're encouraged to change this to whatever value you like the best), all of these modifiers alter the gameplay, so it's recommended you don't start changing the modifiers until you're comfortable with the variants themselves. Games resumed from save codes retain their old settings regardless of what the current modifiers are set to, except for Animation Speed Multiplier and Time Between Moves in Replays, which are always based on their current settings and are not saved by save codes.

The first page of the Modifiers screens

The first page contains modifiers with numeric values - most of these are changed via the plus and minus buttons, while Animation Speed Multiplier is changed by typing a new value into its textbox (1 is default speed, higher numbers are faster, lower numbers are slower... except for 0, which means there's no animation and moves are instant). When it comes to the "Randomly-Placed" modifiers, "holes" are, well, holes in the grid (which effectively act as walls), "block tiles" are tiles that move with the others but can't merge with anything (even themselves), and "slippery tiles" are spaces on the grid that tiles will slide right over (as if the slippery tile isn't there).

The second page of the Modifiers screens

The remaining pages have modifiers that are changed by clicking their respective "Switch this Modifier" buttons. The second page has modifiers that change some more basic elements of the gameplay, the third page has modifiers that alter the grid and movement directions, the fourth and fifth pages are where the most game-changing modifiers are found, and the sixth page lets you add moves that trigger automatically. If you decide to enable either of the fourth page modifiers, you should probably play on a larger grid than you otherwise would! Some modes have special interactions with a couple modifiers, with Negative Tiles in particular having the most special interactions.

Custom Grid

I wonder what that "Activate Custom Grid" button does?

Page 3, by default, only offers a few options for the shape of the grid: square, diamond, checkerboard, hexagonal, triangular (made of hexagons), and 3D/4D... but if you want to customize it further, you can click the "Activate Custom Grid" button. Doing so enables you to fully customize the grid by turning page 3 into two pages: one to customize the grid, one to customize the movement directions.

Ah, so that's what it does!

The grid customizer is the easier of the two custom grid screens to use: you can change the dimensions of the grid, you can switch the grid between square tiles and hexagonal tiles, and you can click on any tile to switch what it starts as: a regular empty tile, a hole in the grid, a box tile, or a slippery tile. You can't make any tile start as a number tile, though, since different modes have different tiles and the custom grid applies to every mode.

The directions customization screen, with the up arrow button selected.

On the directions customization screen, you can alter the direction buttons freely. To alter a specific button, click on that button. Once a button is selected, you can fill in the textboxes to change its attributes. When it comes to movement magnitudes, -1 vertical is up, +1 vertical is down, -1 horizontal is left, +1 horizontal is right, +1 vertical and +1 horizontal is southeast, and so on. +2 vertical is downwards movement that skips every other tile, so tiles jump directly from their current space to the space two tiles below them (These directions are different on a hexagonal grid). The five modifiers with "(%)" next to them are measured in percentage of the arrows box, where 100% is the entire box. To change which keys on the keyboard correspond to each direction, click the "Remove Last Key" and "Add New Key" buttons; the former removes the last keyboard key in the list for that direction. The "Add New Key" button, when clicked, becomes "Listening...", indicating that you should press a key on the keyboard; if the key you clicked isn't already assigned to a direction, then it will be added to the currently selected direction. The "Remove This Direction" button does what it says it does. To un-select a direction, click the direction that's already selected; if there's no direction selected, then you can click the "Add New Direction" button to add a new direction.

Intro to the Custom Game

The 2048 Power Compendium has quite a few modes already, but what if you have an idea for a mode that isn't already in the game? That's what the Custom Game is for! Going through the process of creating a Custom Mode will let you create your own mode that can be played in the 2048 Power Compendium, including a save code you can use to share the mode with others if you wish. In the process of creating a Custom Mode, you'll decide what tiles can spawn, what the allowed merges are, how the tiles are colored, the color of the background elements, and more. Modifiers will work on Custom Modes just as well as they'd work on any other mode, too.

The Custom Mode is designed to allow for the creation of modes where the progression of tiles follows a single cyclic path. This mainly refers to modes in the style of the modes on Page 1, but also includes the first three rows of Page 2. All of these modes can be recreated in the Custom Game, but modes beyond that point are generally beyond what the Custom Game can be used to create. When creating a Custom Mode, each tile has a numeric index: for example, in a 2048 recreation, 1 would be Tile #1, 2 would be Tile #2, 4 would be Tile #3, 8 would be Tile #4, 16 would be Tile #5, 32 would be Tile #6, and so on. It is these index numbers that you have to work with when deciding merges and tile color schemes.

Alright, let's get into explaining each step of the process of creating a Custom Mode.

Spawning Tiles

A replication of 2048's spawning tiles

The first step to making a Custom Mode is deciding what tiles can spawn, and how likely each one is to spawn. For each index starting from 1, you get to choose what the number on that tile is and how likely it is to spawn. Use the "Add a New Spawning Tile" button to add higher indices to the list. A tile's spawn chance determines how likely it is to spawn: a spawning tile with spawn chance 2 is twice as likely to spawn as a spawning tile with spawn chance 1. If, for example, you want Tiles #2 and #3 to spawn but not Tile #1, then just set Tile #1's spawn chance to 0 - if a tile's spawn chance is 0, that tile can't spawn anyway so its value in the list of spawning tiles is ignored. The values of the spawned tiles must be whole numbers, but the values don't have to be positive - tiles whose values are 0 or negative are allowed, though if there are any negative spawning tiles then the Negative Tiles modifier won't work on that Custom Mode since it already has negative tiles.

Normally, tiles spawn with random chance, like in most modes. If you instead want spawning to work the way it does in modes like 1535 1536 1537, where there's a "box" that spawning tiles are pulled from (to ensure a more even spread of spawning tiles), the "Each tile has a random chance to spawn" button can be clicked to change it into (or back from) the box behavior. When the spawning tiles are in box behavior, the spawn chance is replaced with the amount of that tile in the box.

Merges

A replication of 2187's merges

The merges are the most important part of making a Custom Mode. Each merge has a few properties to it: which tile indices the merge can apply to, what the input tiles of the merge are, whether the merge must be ordered, and what the output tile of the merge is. What a merge does with the indices is up to you to decide, but what it does to the tiles' visible numbers is not: the numbers of tiles in a merge are always added, so if a 4 tile and a 5 tile merge, they'll always merge into a 9 tile, regardless of what those tiles indices are internally.

There are two options for deciding how to restrict a merge to certain indices (you can switch between these two forms via the "Change Form" button): either you can outright specify specific indices it works for, or you can make it based on a minimum, maximum, and modulo structure: for example, if the rule is 1 n 30 and n = 2 mod 3, then n must be 2, 5, 8, 11, 14, 17, 20, 23, 26, or 29, i.e. n must be between 1 and 30 and it must be 2 more than a multiple of 3. If you wish for there to be no maximum, leave the maximum box blank (this sets the maximum to Infinity). Note that if you try to enter, say, 5 mod 2, it will automatically be set to the equivalent 1 mod 2, so when editing these boxes you should probably edit the second modulus box before the first one. When specifying specific indices, enter a number into the blank textbox at the end of the list to add a new allowed index, make a textbox blank to remove that allowed index and all of the ones after it.

Merges can be as long or short as you want them to be. One of the tiles in the merge will always be "Tile n", where n is one of the allowed indices for that merge, and you set the other input tiles relative to that one. Input tiles are typically n plus or minus some number, but you can use the "Change Form" button below each input tile to switch it between n + c form and just c, where c is a number. This, for example, lets you make a merge where regardless of what n is, one of the tiles in the merge is a 1 (for 2047-style modes). The output tile also takes such a form.

A merge cannot have more than one output tile, but it can have no output tiles (if you want to make a merge where the colliding tiles are just destroyed). If a merge only has the single "Tile n" input tile, then it becomes more of an "effect" than a merge, an effect that applies to any valid tile at the end of each move (like the decaying in Isotopic 256). If multiple merge rules could apply at once, the merges earlier in the order of merges apply: for example, if you have a mode where both 1+1 and 1+1+1 are valid merges, then you need to put the 1+1+1 merge earlier in the Custom Mode's merge order, otherwise the 1+1 merge will take priority.

If you click the "The input tiles can be in any order" box, it changes to "The input tiles must be in the right order", which means that merge only occurs if the input tiles line up correctly: the first "Tile n" must be the tile colliding into the rest of the line, and so forth from there, with the last input tile being at the end of the line. This can be used to recreate the Bicolor modes, among other things.

Consistency Check

The result of running the consistency check on a 1296 recreation

Aside from the spawning tiles, you don't directly decide what the numbers on the tiles are: the numbers arise naturally as a consequence of the spawning tiles and the merges you choose. The consistency check is a chance for you to confirm that the numbers are doing what you designed the mode to do, i.e. that your mode is working as intended. Pressing "Run the check" will run some code that examines the mode to figure out what the first few tiles' numbers are, and then once you've done the check you can click "View Generated Tiles" to see what the tiles' numbers will be.

The check might come across an "inconsistency" and stop early: this means that, within the merge rules you've set up, it's possible for a tile with the same index to have multiple values, such as if Tile #5 was creatable as both a 6+2 merge and as a 3+9 merge (meaning Tile #5 could be an 8 tile or a 12 tile). This usually isn't something you'd want to have, since a tile's behavior in a Custom Mode is determined only by its index, not by its value. That being said, if you want this behavior, the Custom Mode process won't stop you.

Tile Colors

An example of a color rule

Choosing the colors of tiles is often the hardest part, or at least the longest part, of creating a Custom Mode. Color rules have the same possible restrictions as merges in terms of what tiles they apply to: you can either specify which tiles they apply to directly, or follow a minimum, maximum, modulo rule. As with merges, if multiple color rules could apply to a single tile, the earliest one in the order applies first. Color rules can take a while to set up, so unlike with merges (where the "Add a Merge" button adds a new default merge to the end of the list), the "Add a Color Rule" button adds a duplicate of the color rule you're currently looking at to the end of the color rule list.

For each color in the background of a tile, you get to choose its color via one of the three typical computer color systems: RGB, HSL, or HSV. HSL is used by default, but you can use the "Change Color System" button to switch to one of the other two. The box next to the "Change Color System" button is not clickable: it's a visual aid that shows you the color you currently have selected. The text box next to that lets you enter a hex code for a color, if you'd rather do that than set each of the three aspects separately. The color you choose for a given color rule is technically only the starting color for that rule (where you can choose the start point via the "At Tile #" box), as below the part where you choose the starting color, you can choose how the color varies as the index increases. For each aspect of the color, you can choose whether that aspect linearly increases, or gets exponentially closer to (or further away from) a particular value.

The background of a tile doesn't have to be a single color - it can be a gradient. Use the "Add a Gradient Entry" button to add another color to the gradient. Once a tile's background has more than one color, some additional properties become relevant. The Gradient Position box lets you adjust the order and positioning of each color in the gradient. Gradients come in three types: Linear (the gradient goes from up to down by default. This can be changed via the Gradient Direction box), Radial (the gradient goes from center to outside), and Conic (the gradient starts at the top and loops around in a circular path. The starting angle can be changed). Linear and Radial gradient positions work in percentage (out of 100), but Conic gradient positions work in degrees (out of 360).

The text color of the tile is also part of a color rule. The text can only be one color, not a gradient, but unlike the background, its transparency can be changed: = 1 is fully opaque, = 0 is invisible, and anything in between is partially transparent.

Background

The background color for a 2048 recreation

This screen of the Custom Mode lets you change the colors of five things: the background of the mode while you're playing it, the background of the "mode select" screen for the mode (the screen that lets you change the grid size and such), the color of empty spaces on the grid, the color of the back of the grid (the space between the empty spaces), and the color of the text within the mode (the rules text and such). These colors are chosen in the same way as the tile colors are. The two backgrounds can be gradients, but the empty spaces, grid back, and text color can only be single colors.

Other Stuff

The last Custom Mode screen for a 3125 recreation

The final screen of the Custom Game has a few assorted items that didn't fit anywhere else.

First of all, this is where you decide which tile(s) you need to make to win the mode. These are decided by index, and you can decide how many "winning tile candidates" there are and how many of them you need to discover in order to win. If the Amount Required to Win is N/A, then the mode cannot be won. There's also options for Losing Tiles: if you discover enough of these to hit the Amount Required to Lose, it's game over even if the board hasn't filled up yet. A typical 2048 Power Compendium mode doesn't use this mechanic, but I figured I'd throw it in.

This screen is where you decide the rules text of the mode. This isn't automatically generated - you have to specify it yourself. The first box is where the "heading" of the mode would go if it has one (like "Powers of 4" for 1024), the second box is for the title of the mode, and the third box is for the description of the mode. You do not get to make the text that describes the spawning tiles; that text is automatically generated.

By default, the default size of the grid for a Custom Mode is 2 more than the length of the longest merge in the mode, but you can change that default here if you wish. To accomodate for different grid shapes from the Modifiers, the default grid size must be a square - you can't set the default length and width separately. Of course, as with regular modes, the default grid size is just that: a default, which the player of the mode can still stray away from.

Custom Mode Save Codes

The screen you see upon completion of a Custom Mode

Upon finishing the creation of a Custom Mode, you're given a save code for that mode. You should copy this code and save it somewhere. Later, you can paste this code into the "Resume from Save Code" screen from the main menu to jump into playing that Custom Mode without having to recreate the whole mode, and (by nature of being a string of text) you can share the save code with others if you wish. You can also paste a Custom Mode save code into the screen reached from the "Load from a Save Code" button on the Custom Mode's beginning screen if you wish to edit that Custom Mode.
